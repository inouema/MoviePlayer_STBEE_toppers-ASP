(1) 対応しているターゲットシステムの種類・構成

STBEE(GCC)ターゲット依存部は，STM32F103VB(Cortex-M3)を搭載したストロベリ
ーリナックス社のSTBEEボードをサポートする．プロセッサ依存部として，
ST32F(GCC)プロセッサ依存部をプロセッサ依存部としてARM_Mプロセッサ依存
部を使用する．


(2) 使用する開発環境と動作検証した条件（バージョン，オプション等）

・コンパイラ

GCC 4.4.1  (Sourcery G++ Lite 2010q1-188) で動作確認を行った．

・デバッグ環境

デバッグ環境としては，JTAGデバッガにST-Link，デバッガとしてuVisionを用
いた環境で動作確認を行っている．

動作確認した uVision は以下のバージョンである．

    V4.22.15.0

・DFUによる書き込み

DFUを用いたカーネルの書き込みや実行をサポートしている．DFUを用いる場合
は，Makefile.target の USE_DFU を true に定義すること．

(4) メモリマップ

標準
　内蔵FlashROM
  ・アドレス : 0x08000000
  ・サイズ   : 512KB
  ・配置セクション
   ・vectorセクション
   ・textセクション
   ・rodataセクション
   
  内蔵RAM
  ・アドレス : 0x20000000
  ・サイズ   : 64KB
    ・dataセクション
    ・bssセクション
    
DFU使用時
　内蔵FlashROM
  ・アドレス : 0x08003000
  ・サイズ   : 500KB
  ・配置セクション
   ・vectorセクション
   ・textセクション
   ・rodataセクション
   
  内蔵RAM
  ・アドレス : 0x20000000
  ・サイズ   : 64KB
    ・dataセクション
    ・bssセクション    

(5) シリアルインタフェースドライバの情報

コンソール出力には，USARTのチャネル1を用いる．通信フォーマットは以下の
通りである．

  ・1152200bps, Data 8bit, Parity none, Stop 1bit, Flow control none

(6) システムログ機能の情報

システムログの低レベル出力は，UART0を用いる．通信フォーマットは，
115200bps, 8bit, non-parity, 1topbitである．
